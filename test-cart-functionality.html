Customer Platform Cart Functionality Test

🛒 Cart Features to Test

What to look for in the customer platform:
  1. Cart Button in Header - Should see a cart icon with count badge in the top-right
  2. Add Items to Cart - Click + buttons on products to add items
  3. Cart Count Updates - The red badge should show total items in cart
  4. Cart Sidebar Opens - Click cart button or it should auto-open when first item added
  5. Checkout Button Visible - "Proceed to Checkout" button should be enabled when cart has items
  6. Checkout Flow - Click checkout to open the order form modal

📱 Customer Platform

The customer platform should now have a visible cart button and working checkout flow:

✅ Expected Behavior

✅ Cart Button Visible - Purple gradient button with cart icon in header
✅ Cart Count Badge - Red circle showing number of items (hidden when 0)
✅ Auto-Open Cart - Cart sidebar slides in when first item is added
✅ Checkout Button - "Proceed to Checkout" button enabled when cart has items
✅ Order Form - Modal opens with customer name, phone, payment method fields
✅ Order Submission - Form submits to backend and shows success with order reference

🔧 Troubleshooting

If cart button is not visible:
  • Check browser console for JavaScript errors
  • Refresh the page (Ctrl+F5 or Cmd+Shift+R)
  • Verify the customer platform server is running on port 3001
If checkout doesn't work:
  • Check that backend server is running on port 8000
  • Look for CORS errors in browser console
  • Verify products are loading (should see product cards)

🎯 Quick Test Steps

  1. Load Page - Customer platform should load with products
  2. Find Cart Button - Look for cart icon in top-right of header
  3. Add Item - Click + button on any product
  4. Check Cart Count - Red badge should appear with "1"
